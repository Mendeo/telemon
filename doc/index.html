Telemon: простой мониторинг сервера на Node.js

Назначение

Программа предназначена для мониторинга домашнего сервера.
Отслеживаются различные критические события. При их возникновении происходит отправка сообщения.

На сегодняшний день реализовано отправка его на электронную почту и в телеграм при помощи телеграм бота, а также просто по TCP.

Для телеграм вы создаёте своего телеграм бота, указываете в файле senders/telegram/credentials.json его токен и chat id вашего телеграма (см. senders/telegram/credentials.template.json). Программа будет слать сообщения в этот бот через Telegram API.

Для отправки по электронной почте нужно в senders/email/credentials.json указать адресс отправителя, пароль от почты и адресс назначения и параметры smtp сервера (см. senders/email/credentials.template.json).

Какие именно события отслеживаются определяется пользователем в файле main.mjs.

В этом же файле нужно указать в переменной senders все модули, которые будут использоваться для отправки сообщений. По умолчанию там указан только модуль отправки по электронной почте.

Для отслеживания событий применяются механизм слежения за заданным каталогом или файлом (file_watcher.mjs) или переодическое выполнения заданной команды (command_watcher.mjs), с целью обнаружить важные изменения в её выводе. Есть возможность принимать внешние события (tcp_watcher.mjs). Они будут приходить по TCP и перенаправляться в сендеры (телеграм, электронная почта). Указанные модули наблюдения за системой находятся в папке watchers.

Работа с программой

На сегодняшний день в файле main.mjs заданы:

  • Оповещение при включении сервера.
  • Оповещение от внешнего источника (в моём случае - это проблемы с питанием, для этого отдельная программа следит за питанием и направляет сообщения по TCP)
  • Оповещение о новой системной почте.
  • Оповещения о новых сессиях пользователей.
  • Оповещение об изменении статуса рэйд массива.
  • Оповещение о превышение температуры процессора заданного порога.
  • Оповещение об изменении статуса жёстких дисков (превышение температуры и проверка важных SMART параметров).
  • Оповещение о повышенной нагурзки на ЦП.
  • Оповещение о превышении траффика (последовательно отправляются оповещения о превышении 2ГиБ, 4ГиБ, 6ГиБ, 8ГиБ и более 10ГиБ)

Другие отслеживания пользователь может задать по аналогии. В папке doc представлена автоматически сгененированная документация на используемые модули слежения command_watcher, file_watcher, а также на модули отправки сообщений в телеграм и TCP-сервер.

Запуск

Для тестового запуска используйте команду:

node ./main.mjs

Для автоматического запуска подразумевается работа программы в качестве сервиса systemd. Пример юнита представлен в файле telemon.service
